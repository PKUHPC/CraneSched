# 作业退出代码

`cacct` 命令中的 `EXITCODE` 列记录了用户作业终止的原因。  
例如，在 `0:15` 中，**第一个代码**（`0`）是**主代码**，**第二个代码**（`15`）是**次代码**。

---

## 任务级退出码报告（用于 crun 作业）

对于具有多个任务的交互式 crun 作业（使用 `--ntasks-per-node > 1`），当任何任务失败时，系统会提供详细的退出码信息：

**提供的信息：**
- **任务ID**：标识哪个特定任务失败
- **退出码**：实际的退出码（0-255）或信号编号
- **信号状态**：指示任务是否被信号终止

**触发时机：**
- 当 crun 作业中的任何任务以非零退出码退出时自动触发
- 报告作业中所有任务的退出码，而不仅仅是失败的任务
- 帮助用户快速识别多任务作业中的问题任务

这种细粒度的报告能够更有效地调试和诊断多任务作业故障。

---

## 主代码

程序 `exit` 返回值的值，范围为 0-255

---

## 次代码

程序退出信号：

- 0-63：信号值

Crane 定义的代码：

- 64：已终止
- 65：权限被拒绝
- 66：Cgroup 错误
- 67：文件未找到
- 68：生成进程失败
- 69：超出时间限制
- 70：Crane 守护进程关闭
- 71：执行错误
- 72：RPC 失败

---

## JSON 格式说明

- 值 **0–255** 表示**退出返回值**。  
- 值 **256–320** 表示**程序退出信号**。  
- **320 以上**的值表示 **Crane 定义的代码**。
